<objective>
Create the ChromeSessionManager module with CDP connection to existing Chrome, Chrome launch fallback, profile listing, and clean disconnect. Wire it into Electron's IPC layer so the renderer can control the session lifecycle.

Purpose: This is the foundation for all browser automation. Without a reliable CDP session that preserves the user's Google login, every subsequent phase (automation, image detection, model selection) cannot function. The existing `gensparkPlaywright.js` uses `puppeteer.launch()` which creates a new browser and loses cookies. This plan replaces that with `puppeteer-core.connect()` that attaches to the user's existing Chrome.

Output: Three new files in `electron/chrome-session/` implementing session management, Chrome discovery, and profile reading. Updated `main.js` with new IPC handlers and `preload.js` with new channel exposures.
</objective>

Create three files in `electron/chrome-session/`:

**chromeDiscovery.js** — Chrome executable and process detection:
- `findChromePath()`: Detect Chrome installation via Windows registry (`HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\App Paths\chrome.exe` and `HKCU` variant) using `child_process.execSync('reg query ...')`. Fallback to common paths: `C:\Program Files\Google\Chrome\Application\chrome.exe`, `C:\Program Files (x86)\...`, `%LOCALAPPDATA%\Google\Chrome\Application\chrome.exe`. Return `null` if not found (caller handles manual override).
- `isPortInUse(port, host='127.0.0.1')`: Use `net.Socket` to check if something is listening on the port. 1-second timeout. Returns Promise<boolean>.
- `isChromeRunning()`: Use `child_process.execSync('tasklist /FI "IMAGENAME eq chrome.exe" /NH')` to check if any chrome.exe process exists. Returns boolean. Wrap in try/catch, return false on error.
- `launchChromeWithDebugPort(chromePath, port, debugProfilePath)`: Spawn Chrome with `--remote-debugging-port={port}` and `--user-data-dir={debugProfilePath}`. The `debugProfilePath` should default to `%APPDATA%/AutomatizarAnimacoes/ChromeDebugProfile`. Use `spawn(chromePath, args, { detached: true, stdio: 'ignore' })` and call `child.unref()`. Return the child process reference.
- `waitForPort(port, timeoutMs=15000, intervalMs=500)`: Poll `isPortInUse()` until true or timeout. Returns Promise<void>, rejects on timeout.
- `findFreePort()`: Use `net.createServer()` to find an available port. Bind to port 0, read the assigned port, close the server, return the port.
- `cleanupStaleLocks(profileDir)`: Remove stale Chrome lock files (`lockfile`, `SingletonLock`, `SingletonSocket`, `SingletonCookie`) from the given directory. Only clean if no chrome.exe process is currently using that profile. Silent on errors.

**chromeProfileReader.js** — Chrome profile metadata:
- `listChromeProfiles()`: Read `%LOCALAPPDATA%\Google\Chrome\User Data\Local State` JSON file. Parse `profile.info_cache` to extract profile entries. Return array of `{ profileDir, name, email, gaiaName, isDefault }`. Each entry has `profileDir` (e.g., "Default", "Profile 1"), `name` (user-set name), `email` (Google account email from `user_name`), `gaiaName` (from `gaia_name`), `isDefault` (true if profileDir === "Default"). Return empty array if file doesn't exist or parsing fails. Format display string per user decision: `"email@example.com (Profile Name)"` — store as `displayName` field.
- If `andrewebemp@gmail.com` is found among profiles, flag it with `isPrimary: true` (per user's specific idea about primary account).

**ChromeSessionManager.js** — Session lifecycle with state machine:
- Use `EventEmitter` as base class (extend it). CommonJS module.
- State machine with states: `idle`, `discovering`, `connecting`, `connected`, `reconnecting`, `disposed`.
- `#state`, `#browser`, `#page`, `#port`, `#wsEndpoint` as private fields.
- Constructor accepts `options = { port: 9222, debugProfilePath, chromePath }` with sensible defaults.
- `async connect(profileDir)`:
  1. Assert state is `idle` or `disposed`. Set state to `discovering`.
  2. Call `isPortInUse(port)`. If true, try `puppeteer.connect({ browserURL: 'http://127.0.0.1:{port}', defaultViewport: null })`. If connect succeeds, set `#browser`, emit `connected`. If connect fails (not Chrome on that port), call `findFreePort()` and use that port instead.
  3. If port not in use (no running Chrome with debug): call `isChromeRunning()`. If Chrome IS running (without debug port), throw error with code `CHROME_NO_DEBUG_PORT` and message in Portuguese: "O Chrome está aberto sem a porta de depuração. Por favor, feche o Chrome para que o app possa reiniciá-lo com a depuração habilitada."
  4. If no Chrome running at all: set state to `connecting`. Find Chrome path (auto-detect or from options). Clean stale locks. Launch Chrome with debug port using `launchChromeWithDebugPort()`. Call `waitForPort()`. Connect via `puppeteer.connect({ browserURL })`.
  5. After connecting: find or create Genspark tab (see `getPage` logic below). Minimize Chrome window via CDP `Browser.setWindowBounds`. Set state to `connected`. Emit `connected` event.
- `async getPage()`:
  1. Assert state is `connected`.
  2. If `#page` is still valid (not closed), return it.
  3. Call `browser.pages()`. Find page where URL includes `genspark.ai`. If found, set as `#page` and return.
  4. If no Genspark tab, create new page with `browser.newPage()`, navigate to `https://genspark.ai/agents/image-generator` with `{ waitUntil: 'domcontentloaded', timeout: 30000 }`. Set as `#page`, return.
- `async dispose()`:
  1. If `#browser` exists, call `#browser.disconnect()` (NEVER `#browser.close()`).
  2. Set `#browser = null`, `#page = null`.
  3. Set state to `disposed`. Emit `disposed` event.
- `getState()`: Return current state string.
- `isConnected()`: Return `#state === 'connected'`.
- Private `#minimizeChromeWindow()`: Create CDP session on first page, call `Browser.getWindowForTarget`, then `Browser.setWindowBounds` with `windowState: 'minimized'`. Detach session. Non-critical — catch and log any errors.
- Private `#setState(newState)`: Update state, emit `stateChanged` with `{ from, to }`.
- Private `#assertState(allowedStates)`: Throw if current state not in allowed list.

Use `require('puppeteer-core')` (NOT `require('puppeteer')`). The project has puppeteer-core@24.37.1 installed.

**In electron/main.js**, add new IPC handlers after the existing Genspark section (at the end of the file). Import the new modules:

```javascript
const ChromeSessionManager = require('./chrome-session/ChromeSessionManager');
const { listChromeProfiles } = require('./chrome-session/chromeProfileReader');
```

Create a singleton instance:
```javascript
let chromeSession = null;
```

Add these IPC handlers:

1. `chrome-session-connect`: Accepts `{ profileDir }`. Creates `ChromeSessionManager` if not exists (lazy init). Calls `chromeSession.connect(profileDir)`. Wires event listeners to forward state changes to renderer via `mainWindow.webContents.send('chrome-session-state', ...)`. Returns `{ success: true, state }` or `{ success: false, error, code }`. On `CHROME_NO_DEBUG_PORT` error, return `{ success: false, error: message, code: 'CHROME_NO_DEBUG_PORT' }`.

2. `chrome-session-disconnect`: Calls `chromeSession.dispose()`. Returns `{ success: true }`. Handles case where `chromeSession` is null gracefully.

3. `chrome-session-get-status`: Returns `{ state: chromeSession?.getState() || 'idle', connected: chromeSession?.isConnected() || false }`.

4. `chrome-session-get-profiles`: Calls `listChromeProfiles()`. Returns `{ success: true, profiles }`. This replaces the existing `genspark-get-chrome-profiles` handler functionality but keep the old one working too for backward compatibility.

Wire ChromeSessionManager events to renderer:
- On `stateChanged`: send `chrome-session-state` event with `{ from, to }` to renderer.
- On `connectionLost`: send `chrome-session-connection-lost` to renderer.
- On `reconnecting`: send `chrome-session-reconnecting` with `{ attempt }`.
- On `reconnected`: send `chrome-session-reconnected` with `{ attempt }`.
- On `reconnectExhausted`: send `chrome-session-reconnect-exhausted`.

**In electron/preload.js**, add new API methods in the `electronAPI` object. Add a new section `// === Chrome Session Management ===` after the existing Genspark sections:

```javascript
// === Chrome Session Management ===
chromeSessionConnect: (config) => ipcRenderer.invoke('chrome-session-connect', config),
chromeSessionDisconnect: () => ipcRenderer.invoke('chrome-session-disconnect'),
chromeSessionGetStatus: () => ipcRenderer.invoke('chrome-session-get-status'),
chromeSessionGetProfiles: () => ipcRenderer.invoke('chrome-session-get-profiles'),

// Chrome Session Events
onChromeSessionState: (callback) => {
  ipcRenderer.on('chrome-session-state', (event, data) => callback(data));
},
onChromeSessionConnectionLost: (callback) => {
  ipcRenderer.on('chrome-session-connection-lost', (event, data) => callback(data));
},
onChromeSessionReconnecting: (callback) => {
  ipcRenderer.on('chrome-session-reconnecting', (event, data) => callback(data));
},
onChromeSessionReconnected: (callback) => {
  ipcRenderer.on('chrome-session-reconnected', (event, data) => callback(data));
},
onChromeSessionReconnectExhausted: (callback) => {
  ipcRenderer.on('chrome-session-reconnect-exhausted', (event, data) => callback(data));
},
removeChromeSessionListeners: () => {
  ipcRenderer.removeAllListeners('chrome-session-state');
  ipcRenderer.removeAllListeners('chrome-session-connection-lost');
  ipcRenderer.removeAllListeners('chrome-session-reconnecting');
  ipcRenderer.removeAllListeners('chrome-session-reconnected');
  ipcRenderer.removeAllListeners('chrome-session-reconnect-exhausted');
},
```
