Extend ChromeSessionManager.js (created in Plan 01-01) with reconnection logic:

**Disconnect event handler** — Add a private method `#setupDisconnectHandler()` that:
1. Listens for `browser.on('disconnected', async () => { ... })`.
2. First checks: if `#state === 'disposed'`, this is intentional (user called dispose), so return silently — do NOT reconnect.
3. Otherwise, set state to `reconnecting`. Emit `connectionLost` event (this triggers the "Connection lost, reconnecting..." message in the UI via IPC).
4. Begin retry loop.

Call `#setupDisconnectHandler()` at the end of the `connect()` method, after successful connection.

**Retry loop** — Inside the disconnect handler:
- Define delays array: `[3000, 6000, 12000]` (per locked user decision: 3s, 6s, 12s).
- For each attempt (0, 1, 2):
  1. Emit `reconnecting` event with `{ attempt: attempt + 1, maxAttempts: 3, delayMs: delays[attempt] }`.
  2. Wait `delays[attempt]` ms using a `sleep()` helper (simple `new Promise(resolve => setTimeout(resolve, ms))`).
  3. Try to reconnect: `puppeteer.connect({ browserURL: 'http://127.0.0.1:{port}', defaultViewport: null })`.
  4. If success: set `#browser` to new browser, call `#setupDisconnectHandler()` again (re-register for next disconnect), set `#page = null` (must be re-acquired via `getPage()`), set state to `connected`, emit `reconnected` event with `{ attempt: attempt + 1 }`, return.
  5. If failure: emit `reconnectFailed` event with `{ attempt: attempt + 1, error: err.message }`, continue to next attempt.
- After all 3 attempts fail: set state to `idle` (not `disposed` — the user can try again). Emit `reconnectExhausted` event. Do NOT auto-relaunch Chrome (per user decision: show "Try Again" button, user decides when to retry).

**`getPage()` update** — After reconnection, `#page` is null. Update `getPage()` to handle this:
- If `#page` is null or `#page.isClosed()`, re-discover from `browser.pages()` (same logic as original getPage).

**Login detection** — Add a method `async checkLoginStatus()`:
1. Get the current page via `getPage()`.
2. Make sure the page URL includes `genspark.ai`. If not, navigate to `https://genspark.ai/agents/image-generator` first.
3. Wait briefly for page to settle: `await page.waitForNetworkIdle({ idleTime: 2000, timeout: 10000 }).catch(() => {})`.
4. Evaluate in page context: look for sign-in/login buttons. If any button or link text contains "sign in", "log in", "login" (case-insensitive), return `{ loggedIn: false, reason: 'login_button_found' }`.
5. Look for positive signals: user avatar elements, profile icons (`img[alt*="avatar"]`, `img[alt*="profile"]`, `[class*="avatar"]`, `[class*="user-icon"]`). If found, return `{ loggedIn: true }`.
6. If ambiguous (no clear signal either way), return `{ loggedIn: false, reason: 'no_positive_signal' }` — safe default per research.
7. On any evaluation error, return `{ loggedIn: false, reason: 'detection_failed' }`.

Call `checkLoginStatus()` at the end of `connect()`, after getPage succeeds. Emit `loginStatus` event with the result. If `loggedIn === false`, emit `loginRequired` event with message in Portuguese: "Por favor, faça login no Google na página do Genspark antes de continuar."

**Profile verification** — Add a method `async verifyProfile(expectedProfileDir)`:
1. This is a best-effort check (per research: detecting active profile from CDP is limited).
2. Try to navigate to `chrome://version` via a temporary page, read the profile path, compare with expected. If this fails (which it may on some Chrome versions), log a warning and skip.
3. Alternative approach: Read cookies or local storage to check for a Google account email. Compare with the email associated with `expectedProfileDir` from profile reader.
4. If mismatch detected, emit `profileMismatch` event with `{ expected: expectedProfileDir, actual: detectedProfile }`.
5. If verification is inconclusive, silently continue (do not block).

Add profile verification call in `connect()` after login check, only when connecting to an already-running Chrome (not when we launched it ourselves, since we control the profile in that case).

**Create electron/chrome-session/sessionPersistence.js:**

Uses a simple JSON file at `%APPDATA%/AutomatizarAnimacoes/session-config.json` to persist session preferences.

- `getConfigPath()`: Returns the full path to `session-config.json`. Creates the parent directory (`%APPDATA%/AutomatizarAnimacoes/`) if it doesn't exist.
- `saveLastProfile(profileData)`: Accepts `{ profileDir, email, name }`. Reads existing config (or creates empty object), updates `lastProfile` field with the profile data and a `savedAt` ISO timestamp. Writes back to file. Wrap in try/catch — non-critical, log warning on failure.
- `loadLastProfile()`: Reads config file, returns `lastProfile` object or `null` if file doesn't exist or field is missing. Wrap in try/catch — returns `null` on any error.
- `saveConfig(key, value)`: Generic config setter for future use. Reads existing config, sets `config[key] = value`, writes back.
- `loadConfig(key)`: Generic config getter. Returns `config[key]` or `null`.

**Update ChromeSessionManager.js:**
- Import `{ saveLastProfile, loadLastProfile }` from `./sessionPersistence`.
- In `connect(profileDir)`: After successful connection, call `saveLastProfile({ profileDir, email, name })` with the profile info. The email and name can be retrieved by looking up the profileDir in the profiles list (pass it as part of connect options or resolve it internally).
- Add method `getLastProfile()`: Returns `loadLastProfile()` result. This allows the UI to pre-select the last used profile on launch.
- Wire `getLastProfile()` to a new IPC handler in main.js: `chrome-session-get-last-profile`. Returns `{ success: true, profile }` or `{ success: true, profile: null }`.
- Wire `getLastProfile` to preload.js: `chromeSessionGetLastProfile: () => ipcRenderer.invoke('chrome-session-get-last-profile')`.
